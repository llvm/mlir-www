179 /// Infers the mask type for a transfer op given its vector type and
180 /// permutation map. The mask in a transfer op operation applies to the
181 /// tensor/buffer part of it and its type should match the vector shape
182 /// *before* any permutation or broadcasting. For example,

201 /// Creates a vector select operation that picks values from `newValue` or
202 /// `passthru` for each result vector lane based on `mask`. This utility is used
203 /// to propagate the pass-thru value for masked-out or expeculatively executed
204 /// lanes. VP intrinsics do not support pass-thru values and every mask-out lane
205 /// is set to poison. LLVM backends are usually able to match op + select
206 /// patterns and fold them into a native target instructions.
