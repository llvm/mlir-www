Mapping between SPIR-V storage classes to memref memory spaces.

Note: memref does not have a defined semantics for each memory space; it depends on the context where it is used. There are no particular reasons behind the number assignments; we try to follow NVVM conventions and largely give common storage classes a smaller number. The hope is use symbolic memory space representation eventually after memref supports it.
