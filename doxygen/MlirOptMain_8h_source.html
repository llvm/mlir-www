31 /// Perform the core processing behind `mlir-opt`:
32 /// - outputStream is the stream where the resulting IR is printed.
33 /// - buffer is the in-memory file to parser and process.
34 /// - passPipeline is the specification of the pipeline that will be applied.
35 /// - registry should contain all the dialects that can be parsed in the source.
36 /// - splitInputFile will look for a "-----" marker in the input file, and load
37 /// each chunk in an individual ModuleOp processed separately.
38 /// - verifyDiagnostics enables a verification mode where comments starting with
39 /// "expected-(error|note|remark|warning)" are parsed in the input and matched
40 /// against emitted diagnostics.
41 /// - verifyPasses enables the IR verifier in-between each pass in the pipeline.
42 /// - allowUnregisteredDialects allows to parse and create operation without
43 /// registering the Dialect in the MLIRContext.
44 /// - preloadDialectsInContext will trigger the upfront loading of all
45 /// dialects from the global registry in the MLIRContext. This option is
46 /// deprecated and will be removed soon.
