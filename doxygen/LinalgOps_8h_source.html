47 /// Provide a very simple inference procedure to build the loop ranges from the
48 /// op and its operands. This only works with permutation affine maps and
49 /// patterns of the form `(m, n)[s] -> (m + n - s floordiv 2)`.
50 /// A more advanced Tensor-Comprehension like inference is possible but has
51 /// proven to be ambiguous in unfavorable case.
52 /// As a consequence, we relax the default behavior very conservatively and
53 /// provide an op-specified hook so that Linalg ops may override the behavior.
54 LoopRangeBuilder defaultLoopRangesBuilder(LinalgOp op);

56 /// Returns the name mangled library call name to disambiguate between different
57 /// overloads at the C level. The name mangling scheme is basic and uses MLIR
58 /// type names:
59 /// 1. form a string which is the concatenation of the linalg op name with all
60 /// the operand type names, separate by underscores;
61 /// 2. drop the `linalg.` prefix, and the `<`, `>`, `?` symbols from the type.
62 /// Assumes `op` is a LinalgOp.
63 ///
64 /// Examples:
65 ///
66 /// 1. linalg.fill(%f, %A) : f32, memref<f32>
67 /// name mangles into `linalg_fill_f32_viewf32`
68 ///
69 /// 2. linalg.dot %A, %B, %C :
70 /// (memref<?xf32, stride_specification>,
71 /// memref<?xf32, stride_specification>, memref<f32>)
72 /// name mangles into `linalg_dot_viewxf32_viewxf32_viewf32`
73 ///
74 /// 3. linalg.matmul(...) :
75 /// memref<?x?xf32, stride_specification>,
76 /// memref<?x?xf32, stride_specification>,
77 /// memref<?x?xf32, stride_specification>
78 /// name mangles into `linalg_matmul_viewxxf32_viewxxf32_viewxxf32`
79 std::string generateLibraryCallName(Operation *op);
